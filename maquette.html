📐 Guide de l'architecture Flutter
Écran 1 — Splash

Onboarding + identité visuelle AgroIrri. Présente le projet, le nom de la parcelle, et lance l'initialisation.

Écran 2 — Dashboard

Cœur de l'app : décision du jour, métriques capteurs, jauge humidité sol, prévisions 4 jours.

Écran 3 — Historique

Liste chronologique des décisions ML, volumes irrigués, filtre par mois, statistiques mensuelles.

Écran 4 — Alertes

Notifications push, toggle SMS, aperçu du SMS M. Koffi, alertes sol sec / pluie prévue.

🎯 Packages Flutter recommandés

Pour implémenter cette maquette en vrai code Flutter :

fl_chart http provider flutter_local_notifications shared_preferences flutter_sms intl dio

🔗 Architecture recommandée

Backend : FastAPI Python → endpoints REST
State : Provider ou Riverpod
SMS : Twilio API ou flutter_sms
Stockage : SQLite local + CSV sync
